## tmvisa-mcp

A MCP that allows LLMs to control any instrument on your T&M system 

### Example Configuation
```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "tmvisa-mcp": {
      "type": "local",
      "command": ["/path/to/.venv/bin/python3", "/path/to/server.py"],
      "enabled": true,
      "env": { "PYTHONPATH": "/path/to/tmllm" }
    }
  }
}
```

### visa_list_resources

List all detected VISA instruments on the system. Returns resource names and, when possible,
`*IDN?` identification info (manufacturer, model, serial, firmware).

| Parameter | Type   | Required | Description                          |
|-----------|--------|----------|--------------------------------------|
| query     | string | No       | VISA filter, e.g. `USB?*`, default `?*` |

### visa_open

Open a connection to a VISA instrument by its resource name. Connection is cached and reused.

| Parameter     | Type   | Required | Description                                  |
|---------------|--------|----------|----------------------------------------------|
| resource_name | string | Yes      | VISA resource string, e.g. `TCPIP0::192.168.6.2::gpib0,1::INSTR` |
| timeout       | int    | No       | Timeout in ms, default 5000                  |

### visa_close

Close a connection. If `resource_name` is omitted, closes **all** open instruments.

| Parameter     | Type   | Required | Description                              |
|---------------|--------|----------|------------------------------------------|
| resource_name | string | No       | Resource to close; omit to close all      |

### visa_list_open

List all currently open instrument connections. No parameters.

### visa_send

Send a raw SCPI/vendor command (write, no response expected).

| Parameter     | Type   | Required | Description                              |
|---------------|--------|----------|------------------------------------------|
| resource_name | string | Yes      | Already-opened instrument                 |
| command       | string | Yes      | SCPI command, e.g. `*RST`, `CONF:VOLT:DC` |

### visa_query

Send a SCPI query and return the response string.

| Parameter     | Type   | Required | Description                                    |
|---------------|--------|----------|------------------------------------------------|
| resource_name | string | Yes      | Already-opened instrument                       |
| command       | string | Yes      | SCPI query, e.g. `*IDN?`, `MEAS:VOLT:DC?`      |
| timeout       | int    | No       | Override timeout in ms (0 = use default)        |

### visa_read_raw

Read raw data from the instrument's output buffer without sending a command first.
Useful after `*TRG` + `FETCh?`.

| Parameter     | Type   | Required | Description                    |
|---------------|--------|----------|--------------------------------|
| resource_name | string | Yes      | Already-opened instrument       |

### ieee4882_command

Execute an IEEE 488.2 standard command.

| Parameter     | Type   | Required | Description                                  |
|---------------|--------|----------|----------------------------------------------|
| resource_name | string | Yes      | Already-opened instrument                     |
| command       | string | Yes      | One of the commands listed below              |
| value         | int    | No       | Parameter for \*ESE/\*SRE/\*PSC/\*SAV/\*RCL   |

**Supported commands:**

| Command    | Description                                               | Takes value |
|------------|-----------------------------------------------------------|:-----------:|
| `*IDN?`    | Query identification (manufacturer, model, serial, fw)    |             |
| `*RST`     | Reset to factory defaults                                 |             |
| `*CLS`     | Clear all status registers and error queue                |             |
| `*OPC`     | Set Operation Complete bit when pending commands finish   |             |
| `*OPC?`    | Wait for pending commands, return `1` when complete       |             |
| `*ESE`     | Set/query Standard Event Status Enable register           | 0–255       |
| `*ESR?`    | Query Standard Event Status Register                      |             |
| `*SRE`     | Set/query Service Request Enable register                 | 0–255       |
| `*STB?`    | Read Status Byte register                                 |             |
| `*TST?`    | Self-test, returns `0` (pass) or `1` (fail)               |             |
| `*WAI`     | Wait for all pending operations to complete                |             |
| `*TRG`     | Trigger the instrument                                    |             |
| `*LRN?`    | Return commands needed to restore current state           |             |
| `*OPT?`    | Query installed options                                   |             |
| `*PSC`     | Power-on status clear (`0`=preserve, `1`=clear)           | 0 or 1      |
| `*SAV`     | Save instrument state to internal slot                    | 0–4         |
| `*RCL`     | Recall saved instrument state                             | 0–4         |
| `SYST:ERR?`| Query and clear next error from queue                     |             |

### ieee4882_interpret_register

Interpret `*STB?` or `*ESR?` decimal value into human-readable bit descriptions.

| Parameter     | Type   | Required | Description                             |
|---------------|--------|----------|-----------------------------------------|
| register_type | string | Yes      | `"STB"` or `"ESR"`                      |
| value         | int    | Yes      | Raw decimal value from the instrument   |

**STB (Status Byte) bits:**

| Bit | Value | Meaning                        |
|----:|------:|--------------------------------|
| 2   | 4     | Error queue has entries        |
| 3   | 8     | Questionable data summary      |
| 4   | 16    | Message available in buffer    |
| 5   | 32    | Standard event summary         |
| 6   | 64    | Master summary / SRQ           |
| 7   | 128   | Standard operation summary     |

**ESR (Standard Event Register) bits:**

| Bit | Value | Meaning               |
|----:|------:|-----------------------|
| 0   | 1     | Operation complete    |
| 2   | 4     | Query error           |
| 3   | 8     | Device-specific error |
| 4   | 16    | Execution error       |
| 5   | 32    | Command error         |
| 7   | 128   | Power on              |
